# Skill: Write Specs

## Objective
Transform user ideas into a production-ready system design aligned with Giksoft standards.

## Rules
- Save output to: `production_artifacts/Technical_Specification.md`
- MUST pause for approval
- MUST re-read file if user edits it

## Instructions

1. Analyze the idea deeply

2. Generate a structured spec including:

### Executive Summary

### Functional Requirements

### Non-Functional Requirements
- Performance
- Security
- Scalability

### Architecture

#### Frontend
- React + Vite + Tailwind
- Component-based structure

#### Backend
- Node.js + Express
- Layers:
  - routes/
  - controllers/
  - services/
  - middlewares/

#### Database
- PostgreSQL
- Relational design

### API Design
- REST endpoints
- JWT authentication

### State Management
- React Query or Context

---

3. Save file

4. STOP and ask:

"Do you approve this architecture? You can edit `Technical_Specification.md` and I will update it."
